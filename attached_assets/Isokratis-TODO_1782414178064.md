# Isokratis — Build Task List

Ordered, checkable tasks. **Do them top to bottom.** Each task has a
**Done when** — don't tick it until that's literally true. The goal of the
ordering: reach a *schema-valid* exported Greek law as fast as possible, then
expand.

Priority: `P0` = blocking / do now · `P1` = important · `P2` = later · `P3` = polish.

## PHASE A — Close the validation loop (this is the missing piece)

You have a generator but have never checked its output against the real schema.
Until you do, "it works" is a guess. Fix that first.

### [ ] A1 · `P0` · Get the official AKN 3.0 XSD
- **Why:** It's the only authority on whether your XML is valid.
- **How:**
  - Download the Akoma Ntoso 3.0 schema (the `CSD13` revision, to match the
    LegalParser sample). Source: OASIS LegalDocML GitHub / docs.oasis-open.org.
  - Put it in `schema/akomantoso30.xsd` (plus any imported `.xsd` it references —
    AKN's schema is split across a few files; grab all of them).
- **Done when:** `schema/` contains the XSD set and you can open the root file.

### [ ] A2 · `P0` · Write a validator script
- **Why:** Turns "is this right?" into a precise error list.
- **How:**
  - `pip install lxml` (add to `requirements.txt`).
  - Create `validate.py`:
    ```python
    import sys
    from lxml import etree
    schema = etree.XMLSchema(etree.parse("schema/akomantoso30.xsd"))
    doc = etree.parse(sys.argv[1])
    if schema.validate(doc):
        print("VALID")
    else:
        for e in schema.error_log:
            print(f"line {e.line}: {e.message}")
    ```
- **Done when:** `python validate.py somefile.xml` prints either `VALID` or a list
  of errors.

### [ ] A3 · `P0` · Generate one real sample and validate it
- **Why:** First contact between your renderer and the schema.
- **How:**
  - Write `make_sample.py`: build a `Document` in code with meta + 1 article
    containing 2 paragraphs, render it with `AkomaNtosoRenderer`, write to
    `sample.xml`. (No UI needed — pure script.)
  - Run `python validate.py sample.xml`.
- **Done when:** You have `sample.xml` on disk and a list of validation errors
  (there *will* be some — that's expected and good).

### [ ] A4 · `P0` · Fix validation errors until it passes
- **Why:** A passing baseline is the foundation everything else builds on.
- **How:** Work the error list one at a time. Likely suspects (from code review):
  - `<docType>`/`<docNumber>`/`<docTitle>` directly inside `<p>` in `preface` —
    check the schema's allowed content for `preface`; they may need a different
    container.
  - `<formula name="enactingFormula">` — verify `formula` placement/attributes.
  - The empty-leaf fallback `<content><p/></content>` — confirm `<p/>` empty is
    allowed, or put a non-breaking space / use `<authorialNote>`-free minimal text.
  - Element **ordering** inside `meta` (AKN enforces order:
    identification → publication → … → references → notes). Reorder if flagged.
  - `eId` format on `eventRef`/`keyword` (must be valid NCName-ish per [N3]).
- **Done when:** `python validate.py sample.xml` prints `VALID`.

### [ ] A5 · `P1` · Save the passing sample as a golden file
- **Why:** Regression safety — you'll know instantly if a future change breaks
  validity.
- **How:** Copy the validated `sample.xml` to `tests/golden/nomos_min.akn.xml`.
- **Done when:** The golden file exists and validates.

## PHASE C — Harden the renderer

### [ ] C1 · `P1` · Switch XML generation from strings to `lxml`
- **Why:** Hand-built strings can emit malformed XML (unescaped chars, mismatched
  tags). `lxml` makes that structurally impossible and gives clean serialization.
- **How:** Rebuild `xml_renderer.py` using `etree.Element`/`SubElement` instead of
  string concatenation. Keep the same structure and `§` comments. Serialize with
  `etree.tostring(root, pretty_print=True, xml_declaration=True, encoding="UTF-8")`.
- **Done when:** The lxml renderer produces a file that still validates, and the
  golden test still passes.
- **Note:** Do this *after* A4 passes, so you're refactoring known-good output.

### [ ] C2 · `P1` · Audit escaping & edge cases
- **How:** Test with nasty input: an article containing `&`, `<`, `»`, quotes,
  Greek polytonic, an empty body, a node with children *and* content.
- **Done when:** All produce valid XML (lxml from C1 mostly solves this).

### [ ] C3 · `P2` · Decide `εδάφιο` / `περίπτωση` modelling and apply consistently
- **Why:** Your doc lists `subparagraph` vs `list/point` as an open choice (§11).
  Pick one, document it, use it everywhere.
- **Done when:** One mapping is chosen, noted in the mapping doc's changelog, and
  the renderer follows it.

## PHASE D — Fill the real gaps (highest-value features)

### [ ] D1 · `P1` · Inline references (`<ref>` / mentions)
- **Why:** "Mentions" — the thing you said only LEOS does well. Maps to §5.7.
- **How:**
  - In a node's text, detect/store references to other provisions
    (e.g. "άρθρο 5 του ν. 4330/2015").
  - Render them as `<ref href="/gr/act/2015/4330/...">…</ref>`.
  - Declare targets in the `<references>`/TLC register.
  - Start manual (the user marks a reference in the UI) before any auto-detection.
- **Done when:** A document with one cross-reference exports valid AKN containing
  a resolvable `<ref>`.

### [ ] D2 · `P2` · Amendments / consolidation (`textualMod`)
- **Why:** "When was this article modified, what did it say before" — your core
  value proposition. Maps to §5.9. Hardest; do it last.
- **How:**
  - Model an amendment as: type (substitution/insertion/repeal), source ref,
    destination ref, old text, new text.
  - Render under `<passiveModifications>` with `<textualMod>` →
    `<source>` (exactly **one** `<ref>`), `<destination>`, `<old>`/`<new>`.
  - **Carry-over bug to avoid:** do NOT nest `<ref>` inside `<ref>` in `<source>`
    (the TS version did this).
- **Done when:** A document records one amendment and exports valid AKN with a
  well-formed `<textualMod>`.

### [ ] D3 · `P3` · (Optional) AKN import (reverse renderer)
- **Why:** Only if you want "open an existing `.akn.xml`" / AKN-as-native format.
- **How:** Parse AKN with `lxml`, walk it back into your node model. It's the
  mirror of the renderer.
- **Done when:** `render(import(file)) == file` (round-trips) for the golden file.
- **Note:** Genuinely hard. Skip unless you specifically need it.
